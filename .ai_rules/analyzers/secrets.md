# AI & Developer Guide: secrets.py
이 파일은 파이썬 소스 코드의 개별 라인을 스캔하여 정규 표현식을 바탕으로 하드코딩된 API Key, 비밀번호, 토큰, 비밀키(Private Key) 등 민감한 자격 증명 정보를 탐지하는 경량 시크릿 스캔 기능을 담당합니다.

## 🤖 AI Agent Guidelines
- **함수 시그니처 및 반환 타입**:
  - `find_potential_secrets(lines: list[str]) -> list[SecretFinding]` 함수 형태와 리턴 타입을 엄격히 준수해야 합니다.
  - 탐지된 각 시크릿 후보는 반드시 스키마의 `SecretFinding` 규격으로 반환되어야 합니다.
- **데이터 보안성 유지**: 탐지된 시크릿이 보고서나 로그 등에 그대로 평문 노출되는 일을 막기 위해, 시크릿 값의 일부만 노출하고 나머지는 마스킹 처리하는 `_preview(value: str, visible: int = 4) -> str` 헬퍼 함수를 반드시 적용하여 데이터를 정제하십시오.
- **안티 패턴**: 
  - 외부의 무겁고 설치가 까다로운 시크릿 탐지 라이브러리(예: `detect-secrets`, `git-secrets` 등)를 파이프라인의 필수 종속성으로 강제 결합하지 마십시오.
  - 가벼운 정규 표현식 검색 흐름을 어기고 파일 시스템 전체를 검색하거나 외부 API에 키를 검증 요청하지 마십시오.

## 👨💻 Developer Context
- **오탐(FP) 관리 방향성**: 이 시크릿 탐지는 정규 표현식 기반의 단순 매칭 방식으로 작동하므로 가짜 시크릿이나 무해한 테스트 변수 할당에 대해 다량의 오탐(False Positive)이 발생할 수 있습니다. 룰을 너무 빡빡하게 제한하기보단 "가능성 있는 후보군"을 발굴해 static signal로 등록해 두고, 최종 유효 여부는 LLM과 개발자가 2차로 분석할 수 있도록 설계 방향을 조율하십시오.
- **테스트 케이스 주의사항**: 시크릿 탐지 기능을 테스트할 때 실제 운영에 쓰이는 진짜 Private Key나 API Key를 `tests/`나 코드 내에 하드코딩하여 반영하지 마십시오. 반드시 Mock 값과 더미 문자열을 활용해 테스트하십시오.
