# AI & Developer Guide: cli.py
이 파일은 프로젝트의 커맨드라인 인터페이스(CLI) 진입점을 처리하며, 사용자 입력을 받아 분석 경로를 파악하고, 필요한 설정값과 인자를 파이프라인에 전달하여 결과를 터미널에 출력하는 역할을 담당합니다.

## 🤖 AI Agent Guidelines
- **진입점의 슬림화**: CLI 모듈은 비즈니스 로직을 직접 수행하지 않고 오직 아규먼트 파싱(`build_parser`)과 파이프라인 호출(`main`) 역할에만 집중해야 합니다.
- **인터페이스 고정**: CLI 옵션(`analyze` 서브커맨드, `--out`, `--max-lines`, `--format`, `--review-client`, `--no-ruff` 등)을 임의로 제거하거나 이름을 수정해서는 안 됩니다. 옵션 변경이 필요할 때는 반드시 `schemas.py` 및 `pipeline.py`와의 호환성을 사전 검증해야 합니다.
- **반환 규격**: `main` 함수는 프로세스의 성공/실패 여부를 운영체제에 명확히 알릴 수 있도록 `int` 타입의 종료 코드를 반드시 리턴해야 합니다.
- **임포트 제약**: 파이프라인이나 분석 클라이언트의 클래스를 동적으로 생성하거나 참조할 때 순환 참조가 발생하지 않도록 주의하고, 필요한 종속성만 명시적으로 임포트하여 사용하십시오.

## 👨💻 Developer Context
- **인수 전달 체계 유지**: CLI 옵션 중 `--review-client`는 `mock`과 `local` 클라이언트를 적절히 분기하여 `Phase1Pipeline` 인스턴스에 주입하도록 설계되어 있습니다. 이 연결 구조가 깨지지 않도록 신규 옵션 추가 시에도 일관된 주입 패턴을 유지하십시오.
- **출력 정합성**: CLI 최종 출력(터미널 print문) 시, 생성된 HTML/JSON/MD 보고서의 절대 경로와 분석된 대상의 경로 정보가 사용자에게 보기 좋고 명확하게 출력되는 구조를 보존하십시오.
